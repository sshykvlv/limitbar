## Принятые решения (не пересматривать)

- **Строки меню без иконок/плашек** — аккаунт первой строкой (главный сканируемый признак при нескольких аккаунтах одного сервиса), «Сервис · План» второй. Email — только в hover-тултипе
- **Вторая строка НЕЙТРАЛЬНЫМ цветом** (secondaryLabelColor), не брендовым тинтом: сервис читается словом, смысловой цвет несут кольца (владелец перерешал поверх моего тинта, коммит 5cc3484 — действует ЭТО)
- **Менюбар-иконка: заполнение = израсходовано, без цифры** — столбики сами показывают статус каждой модели (владелец отменил вариант «остаток + % худшего», коммит 5cc3484 — действует ЭТО)
- **Ховер строк — нативная синяя подсветка**: rounded-rect 5pt инсет, `selectedContentBackgroundColor` (следует за акцентом юзера), текст/кольца адаптируются в белый
- **Иконка приложения** — генерится кодом (`scripts/gen-appicon.swift`), не бинарник из редактора; в самом менюбаре живёт динамический IconRenderer, статическая иконка — только Dock/Finder/лендинг
- **Лендинг: структура Lidless, айдентика своя** (graphite+amber); в навбаре лендинга — глиф «три бара» + wordmark; canonical временно `limitbar.app`
- **Данные и приватность (для FAQ/копирайта, проверено по коду):** официальные usage-эндпоинты с собственными токенами; поллинг НЕ жжёт квоту (метаданные, не модельный вызов); токены в Keychain
- **Распространение как Lidless:** публичный GitHub + MIT + нотаризованный zip напрямую (не App Store — читает локальный ~/.codex, песочница запрещает)
- Тон коммуникации продукта — английский (лендинг/README), внутренние коммиты — русский с «почему»
